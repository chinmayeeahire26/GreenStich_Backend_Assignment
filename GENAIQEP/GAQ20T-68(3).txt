New Code with Fix:  
java
public void loginToFlipkart(String username, String password) {
    WebDriver driver = null;
    try {
        driver = new ChromeDriver();
        driver.get("https://www.flipkart.com");

        // Wait for username field presence
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        WebElement usernameField = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("username")));
        usernameField.clear();
        usernameField.sendKeys(username);  // Dynamic input

        WebElement passwordField = driver.findElement(By.id("password"));
        passwordField.clear();
        passwordField.sendKeys(password);  // Dynamic input

        WebElement loginButton = driver.findElement(By.id("loginBtn"));
        loginButton.click();

        // Verify login success by checking presence of user profile element or logout button
        boolean loginSuccess = false;
        try {
            wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("userProfile")));
            loginSuccess = true;
        } catch (TimeoutException e) {
            // Login failed or user profile not visible
            loginSuccess = false;
        }

        if (!loginSuccess) {
            // Optionally capture screenshot or log failure
            System.out.println("Login failed for user: " + username);
        } else {
            System.out.println("Login successful for user: " + username);
        }

        // Dismiss login popup if present
        try {
            WebElement popupClose = driver.findElement(By.id("popupClose"));
            popupClose.click();
        } catch (NoSuchElementException e) {
            // Popup not present, continue
        }

    } catch (Exception e) {
        System.err.println("Exception during login test: " + e.getMessage());
        e.printStackTrace();
    } finally {
        if (driver != null) {
            driver.quit();
        }
    }
}


Changes Made:  
- Replaced hardcoded credentials with method parameters `username` and `password` to allow dynamic test data input.  
- Added explicit waits (`WebDriverWait`) to ensure elements are present before interaction, improving reliability.  
- Added verification logic after login click to check for a user profile element indicating successful login.  
- Added handling for login failure scenario with appropriate logging.  
- Wrapped the entire test in try-catch-finally to handle unexpected exceptions and ensure driver quits properly.  
- Retained popup dismissal logic with error handling.

New Code with Fix:

public User loginUser() {
    Authentication auth = SecurityContextHolder.getContext().getAuthentication();
    if (auth == null || !auth.isAuthenticated()) {
        // Log or handle unauthenticated access
        System.err.println("Authentication is null or not authenticated");
        return null;  // Or throw custom exception
    }

    String userName = auth.getName();
    Optional<User> userOpt = userRepo.findByEmail(userName);
    if (!userOpt.isPresent()) {
        // Log user not found
        System.err.println("User not found for email: " + userName);
        return null;  // Or throw custom exception
    }

    return userOpt.get();
}


Changes Made:  
- Added null check and authentication status check on `auth` to prevent NullPointerException.  
- Changed `userRepo.findByEmail` to return `Optional<User>` and added presence check to handle user not found scenario gracefully.  
- Added logging for error conditions to aid debugging and UI consistency.  
- Returned `null` or could throw a custom exception depending on application design to handle missing user or unauthenticated access safely.
